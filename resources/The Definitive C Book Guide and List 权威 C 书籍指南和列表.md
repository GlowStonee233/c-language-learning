This question attempts to collect a community-maintained list of **_quality_** books on the [c](/questions/tagged/c "show questions tagged 'c'") programming language, targeted at various skill levels.  
这个问题试图收集社区维护的关于 [c](/questions/tagged/c "show questions tagged 'c'") 编程语言的**_优质_**书籍列表，针对不同的技能水平。

C is a complex programming language that is difficult to pick up on-the-go by reading online tutorials. A comprehensive book is often the best way to learn the language, and finding a good book is the first step. It is important to avoid badly-written books, and even more importantly, books that contain serious technical errors.  
C 是一种复杂的编程语言，很难通过阅读在线教程随时随地掌握。一本综合性的书往往是学习语言的最佳方式，而找到一本好书是第一步。重要的是要避免写得不好的书，更重要的是避免包含严重技术错误的书。

Please suggest edits to the accepted answer to add quality books, with an approximate skill level and a short blurb/description about each book. (_Note that the question is locked, so no new answers will be accepted. A single answer is being maintained with the list_)  
请建议对接受的答案进行编辑，以添加高质量的书籍，并提供大致的技能水平和关于每本书的简短简介 / 描述。（ _请注意，该问题已锁定，因此不会接受任何新答案。列表正在维护一个答案_ ）

Feel free to debate book choices, quality, headings, summaries, skill levels, and anything else you see that is wrong. Books that are deemed satisfactory by the C community here will stick around on the list; the rest will be regularly removed.  
随意讨论书籍选择、质量、标题、摘要、技能水平以及您认为错误的任何其他内容。这里的 C 社区认为令人满意的书籍将留在名单上; 其余的将定期删除。

For books that have reviews by the Association of C and C++ Users (ACCU), a link to those reviews should be added along with the book.  
对于有 C 和 C++ 用户协会 （ACCU） 评论的书籍，应在书籍中添加指向这些评论的链接。

Reference (All Levels)  参考（所有级别）
================================

*   [The C Programming Language (2nd Edition)](https://rads.stackoverflow.com/amzn/click/com/0131103628) - [Brian W. Kernighan](https://en.wikipedia.org/wiki/Brian_Kernighan) and [Dennis M. Ritchie](https://en.wikipedia.org/wiki/Dennis_Ritchie) (1988). It is still a good, short, but complete, introduction to C ([C89](https://en.wikipedia.org/wiki/ANSI_C#C89), not [C99](https://en.wikipedia.org/wiki/C99) or later versions), written by the inventor of C. However, the language has changed and good C style has developed in the last 25 years, and there are parts of the book that show its age.  
    [C 编程语言（第 2 版）](https://rads.stackoverflow.com/amzn/click/com/0131103628)- [Brian W. Kernighan](https://en.wikipedia.org/wiki/Brian_Kernighan) 和 [Dennis M. Ritchie](https://en.wikipedia.org/wiki/Dennis_Ritchie) （1988）。它仍然是一本很好的、简短但完整的 C 语言（[C89](https://en.wikipedia.org/wiki/ANSI_C#C89)，而不是 [C99](https://en.wikipedia.org/wiki/C99) 或更高版本）的介绍，由 C 的发明者编写。然而，在过去的 25 年里，语言发生了变化，发展了良好的 C 风格，并且这本书的某些部分显示出它的年代久远。
    
*   [C: A Reference Manual (5th Edition)](https://rads.stackoverflow.com/amzn/click/com/013089592X) - Samuel P. Harbison and Guy R. Steele (2002). An excellent reference book on C, up to and including C99. It is not a tutorial, and probably unfit for beginners. It's great if you need to write a compiler for C, as the authors had to do when they started.  
    [C：参考手册（第 5 版）](https://rads.stackoverflow.com/amzn/click/com/013089592X)- Samuel P. Harbison 和 Guy R. Steele （2002）。一本关于 C 的优秀参考书，包括 C99。它不是教程，可能不适合初学者。如果您需要为 C 编写编译器，那就太好了，就像作者在开始时必须做的那样。
    
*   [C Pocket Reference (O'Reilly)](https://rads.stackoverflow.com/amzn/click/com/0596004362) - Peter Prinz and Ulla Kirch-Prinz (2002).  
    [C 袖珍参考 （O'Reilly）](https://rads.stackoverflow.com/amzn/click/com/0596004362) - Peter Prinz 和 Ulla Kirch-Prinz （2002）。
    
*   [The comp.lang.c FAQ](http://c-faq.com/) - Steve Summit. Web site with answers to many questions about C.  
    [comp.lang.c 常见问题解答](http://c-faq.com/) - Steve Summit. 网站回答了有关 C.
    
*   Various versions of the C language standards can be found [here](http://www.iso-9899.info/wiki/The_Standard). There is an online version of the [draft C11 standard](http://port70.net/%7Ensz/c/c11/n1570.html).  
    可以在[此处](http://www.iso-9899.info/wiki/The_Standard)找到各种版本的 C 语言标准。[C11 标准草案](http://port70.net/%7Ensz/c/c11/n1570.html)有一个在线版本。
    
*   [The new C standard - an annotated reference](http://www.knosof.co.uk/cbook/cbook.html) (Free PDF) - Derek M. Jones (2009). The "new standard" referred to is the old C99 standard rather than C11.  
    [新的 C 标准 - 带注释的参考（](http://www.knosof.co.uk/cbook/cbook.html) 免费 PDF） - Derek M. Jones （2009）。所指的 “新标准” 是旧的 C99 标准，而不是 C11。
    
*   [Rationale for C99 Standard](http://www.open-std.org/JTC1/SC22/WG14/www/C99RationaleV5.10.pdf).  
    [C99 标准的基本原理](http://www.open-std.org/JTC1/SC22/WG14/www/C99RationaleV5.10.pdf) 。

Intermediate  中间
================

*   [Modern C](https://modernc.gforge.inria.fr/) — Jens Gustedt (2017 1st Edn; 2019 2nd Edn). Covers C in 5 levels (encounter, acquaintance, cognition, experience, ambition) from beginning C to advanced C. It covers C11 and C17, including threads and atomic access, which few other books do. Not all compilers recognize these features in all environments.  
    [现代 C](https://modernc.gforge.inria.fr/) — Jens Gustedt（2017 年第 1 版; 2019 年第 2 版）。涵盖从初级 C 到高级 C 的 5 个级别（相遇、相识、认知、经验、野心）的 C。它涵盖了 C11 和 C17，包括线程和原子访问，这是其他书籍很少有的。并非所有编译器都能在所有环境中识别这些功能。
    
*   [C Interfaces and Implementations](http://www.cs.princeton.edu/software/cii/) - David R. Hanson (1997). Provides information on how to define a boundary between an interface and implementation in C in a generic and reusable fashion. It also demonstrates this principle by applying it to the implementation of common mechanisms and data structures in C, such as lists, sets, exceptions, string manipulation, memory allocators, and more. Basically, Hanson took all the code he'd written as part of building [Icon](http://www.cs.arizona.edu/icon) and [lcc](http://www.cs.princeton.edu/software/lcc/) and pulled out the best bits in a form that other people could reuse for their own projects. It's a model of good C programming using modern design techniques (including [Liskov](https://en.wikipedia.org/wiki/Barbara_Liskov#Recognition_and_awards)'s data abstraction), showing how to organize a big C project as a bunch of useful libraries.  
    [C 接口和实现](http://www.cs.princeton.edu/software/cii/) - David R. Hanson （1997）。提供有关如何以通用且可重用的方式在 C 中定义接口和实现之间的边界的信息。它还通过将其应用于 C 中常见机制和数据结构的实现来证明这一原则，例如列表、集合、异常、字符串作、内存分配器等。基本上，Hanson 将他编写的所有代码作为构建 [Icon](http://www.cs.arizona.edu/icon) 和 [lcc](http://www.cs.princeton.edu/software/lcc/) 的一部分，并以其他人可以在自己的项目中重用的形式提取出最好的部分。这是一个使用现代设计技术（包括 [Liskov](https://en.wikipedia.org/wiki/Barbara_Liskov#Recognition_and_awards) 的数据抽象）进行良好 C 编程的模型，展示了如何将一个大型 C 项目组织为一堆有用的库。
    
*   [The C Puzzle Book](https://rads.stackoverflow.com/amzn/click/com/0201604612) - Alan R. Feuer (1998)  
    [C 拼图书](https://rads.stackoverflow.com/amzn/click/com/0201604612) - Alan R. Feuer （1998）
    
*   [The Standard C Library](https://rads.stackoverflow.com/amzn/click/com/0131315099) - P.J. Plauger (1992). It contains the complete source code to an implementation of the C89 standard library, along with extensive discussions about the design and why the code is designed as shown.  
    [标准 C 库](https://rads.stackoverflow.com/amzn/click/com/0131315099) - PJ Plauger （1992）。它包含 C89 标准库实现的完整源代码，以及有关设计以及代码设计如图所示的广泛讨论。
    
*   [21st Century C: C Tips from the New School](https://rads.stackoverflow.com/amzn/click/com/1449327141) - Ben Klemens (2012). In addition to the C language, the book explains gdb, valgrind, autotools, and git. The comments on style are found in the last part (Chapter 6 and beyond).  
    [21 世纪 C：来自新学校的 C 技巧](https://rads.stackoverflow.com/amzn/click/com/1449327141) - Ben Klemens （2012）。除了 C 语言，书中还解释了 gdb、valgrind、autotools 和 git。关于风格的评论可以在最后一部分（第 6 章及以后）找到。
    
*   [Algorithms in C](https://rads.stackoverflow.com/amzn/click/com/0201314525) - Robert Sedgewick (1997). Gives you a real grasp of implementing algorithms in C. Very lucid and clear; will probably make you want to throw away all of your other algorithms books and keep this one.  
    [C 算法](https://rads.stackoverflow.com/amzn/click/com/0201314525) - 罗伯特 · 塞奇威克 （1997）。让您真正掌握用 C 语言实现算法。非常清晰明了; 可能会让你想扔掉所有其他算法书并保留这本书。
    

*   [Pointers on C](https://rads.stackoverflow.com/amzn/click/com/0673999866) - Kenneth Reek (1997).  
    [C 上的指针](https://rads.stackoverflow.com/amzn/click/com/0673999866) - 肯尼思 · 里克 （1997）。

*   [Problem Solving and Program Design in C (6th Edition)](https://rads.stackoverflow.com/amzn/click/com/0321535421) - Jeri R. Hanly and Elliot B. Koffman (2009).  
    [C 语言问题解决和程序设计（第 6 版）](https://rads.stackoverflow.com/amzn/click/com/0321535421)- Jeri R. Hanly 和 Elliot B. Koffman （2009）。
    
*   [Data Structures - An Advanced Approach Using C](https://rads.stackoverflow.com/amzn/click/com/B00101Y58Q) - Jeffrey Esakov and Tom Weiss (1989).  
    [数据结构 - 使用 C 的高级方法](https://rads.stackoverflow.com/amzn/click/com/B00101Y58Q) - Jeffrey Esakov 和 Tom Weiss （1989）。
    
*   [C Unleashed](https://rads.stackoverflow.com/amzn/click/com/0672318962) - Richard Heathfield, Lawrence Kirby, et al. (2000). Not ideal, but it is worth intermediate programmers practicing problems written in this book. This is a good cookbook-like approach suggested by comp.lang.c contributors.  
    [C 释放](https://rads.stackoverflow.com/amzn/click/com/0672318962) - 理查德 · 希斯菲尔德、劳伦斯 · 柯比等人 （2000）。不理想，但值得中级程序员练习本书中写的问题。这是 comp.lang.c 贡献者建议的一种很好的类似食谱的方法。
    

*   [Object-oriented Programming with ANSI-C](http://www.planetpdf.com/codecuts/pdfs/ooc.pdf) (Free PDF) - Axel-Tobias Schreiner (1993). The code gets a bit convoluted. If you want C++, use C++. It only uses C90, of course.  
    [使用 ANSI-C 进行面向对象编程](http://www.planetpdf.com/codecuts/pdfs/ooc.pdf) （免费 PDF）- Axel-Tobias Schreiner （1993）。代码变得有点复杂。如果需要 C++，请使用 C++。当然，它只使用 C90。

Expert  专家
==========

    [专家 C 编程：深层 C 秘密](https://rads.stackoverflow.com/amzn/click/com/0131774298) - Peter van der Linden （1994）。Sun 编译器团队提供了许多有趣的信息和战争故事，但有些地方有点过时。

    [UNIX 环境中的高级编程](https://rads.stackoverflow.com/amzn/click/com/0321637739) - Richard W. Stevens 和 Stephen A. Rago （2013）。全面描述了如何从 C 代码中使用 Unix API，但不太介绍 C 编码的机制。

Uncategorized  未分类
==================

*   [Essential C](http://cslibrary.stanford.edu/101/EssentialC.pdf) (Free PDF) - Nick Parlante (2003). Note that this describes the C90 language at several points (_e.g._, in discussing `//` comments and placement of variable declarations at arbitrary points in the code), so it should be treated with some caution.  
    [Essential C](http://cslibrary.stanford.edu/101/EssentialC.pdf)（免费 PDF）- Nick Parlante （2003）。请注意，这在几个点上描述了 C90 语言（ _例如_ ，在讨论注释和变量声明在代码中任意点的位置时），因此应该谨慎对待它。
    
*   [C Programming FAQs: Frequently Asked Questions](https://rads.stackoverflow.com/amzn/click/com/0201845199) - Steve Summit (1995). This is the book of the web site listed earlier. It doesn't cover C99 or the later standards.  
    [C 编程常见问题解答：常见问题](https://rads.stackoverflow.com/amzn/click/com/0201845199) - Steve Summit （1995）。这是前面列出的网站的书。它不包括 C99 或更高版本的标准。
    
*   [C in a Nutshell](https://rads.stackoverflow.com/amzn/click/com/0596006977) - Peter Prinz and Tony Crawford (2005). Excellent book if you need a reference for C99.  
    [简而言之 C](https://rads.stackoverflow.com/amzn/click/com/0596006977) - Peter Prinz 和 Tony Crawford （2005）。如果您需要 C99 的参考资料，这是一本很棒的书。
    
*   [C in a Nutshell (2nd Ed.)](https://rads.stackoverflow.com/amzn/click/com/1491904755) - Peter Prinz and Tony Crawford (2016), a reference-style book covering C11.  
    [C 简而言之（第二版）](https://rads.stackoverflow.com/amzn/click/com/1491904755)- Peter Prinz 和 Tony Crawford （2016），一本涵盖 C11 的参考书。
    
*   [Functional C](https://rads.stackoverflow.com/amzn/click/com/0201419505) - Pieter Hartel and Henk Muller (1997). Teaches modern practices that are invaluable for low-level programming, with concurrency and modularity in mind.  
    [功能 C](https://rads.stackoverflow.com/amzn/click/com/0201419505) - Pieter Hartel 和 Henk Muller （1997）。教授对低级编程非常宝贵的现代实践，并考虑到并发性和模块化。
    
*   [The Practice of Programming](https://rads.stackoverflow.com/amzn/click/com/020161586X) - Brian W. Kernighan and Rob Pike (1999). A very good book to accompany K&R. It uses C++ and Java too.  
    [编程实践](https://rads.stackoverflow.com/amzn/click/com/020161586X) - Brian W. Kernighan 和 Rob Pike （1999）。一本非常好的书，可以搭配 K&R。它也使用 C++ 和 Java。
    

*   [C Traps and Pitfalls](https://rads.stackoverflow.com/amzn/click/com/0201179288) by A. Koenig (1989). Very good, but the C style pre-dates standard C, which makes it less recommendable these days.  
    [C 陷阱和陷阱](https://rads.stackoverflow.com/amzn/click/com/0201179288) 作者：A. Koenig （1989）。非常好，但 C 风格早于标准 C，这使得它现在不太值得推荐。
    
    Some have argued for the removal of 'Traps and Pitfalls' from this list because it has trapped some people into making mistakes; others continue to argue for its inclusion. Perhaps it should be regarded as an 'expert' book because it requires moderately extensive knowledge of C to understand what's changed since it was published.  
    一些人主张从这个列表中删除 “陷阱和陷阱”，因为它让一些人犯了错误; 其他人继续主张将其纳入其中。也许它应该被视为一本“专家” 书籍，因为它需要对 C 有一定程度的了解才能理解自出版以来发生了什么变化。
    

*   [MISRA-C](http://www.misra-c.com/Activities/MISRAC/tabid/160/Default.aspx) - industry standard published and maintained by the Motor Industry Software Reliability Association. Covers C89 and C99.  
    [MISRA-C](http://www.misra-c.com/Activities/MISRAC/tabid/160/Default.aspx) - 由汽车行业软件可靠性协会发布和维护的行业标准。涵盖 C89 和 C99。
    
    Although this isn't a book as such, many programmers recommend reading and implementing as much of it as possible. MISRA-C was originally intended as guidelines for safety-critical applications in particular, but it applies to any area of application where stable, bug-free C code is desired (who doesn't want fewer bugs?). MISRA-C is becoming the de facto standard in the whole embedded industry and is getting increasingly popular even in other programming branches. There are (at least) three publications of the standard (1998, 2004, and the current version from 2012). There is also a MISRA Compliance Guidelines document from 2016, and MISRA C:2012 Amendment 1 — Additional Security Guidelines for MISRA C:2012 (published in April 2016).  
    虽然这不是一本书，但许多程序员建议尽可能多地阅读和实现它。MISRA-C 最初旨在作为安全关键型应用程序的指南，但它适用于需要稳定、无错误的 C 代码的任何应用程序领域（谁不想要更少的错误？MISRA-C 正在成为整个嵌入式行业的事实标准，甚至在其他编程分支中也越来越受欢迎。该标准（至少）有三个出版物（1998 年、2004 年和 2012 年的当前版本）。还有一份 2016 年的 MISRA 合规指南文件，以及 MISRA C：2012 修正案 1 — MISRA C：2012 的附加安全指南（2016 年 4 月发布）。
    
    Note that some of the strictures in the MISRA rules are not appropriate to every context. For example, directive 4.12 states "Dynamic memory allocation shall not be used". This is appropriate in the embedded systems for which the MISRA rules are designed; it is not appropriate everywhere. (Compilers, for instance, generally use dynamic memory allocation for things like symbol tables, and to do without dynamic memory allocation would be difficult, if not preposterous.)  
    请注意，MISRA 规则中的一些限制并不适用于所有情况。例如，指令 4.12 规定 “不得使用动态内存分配”。这适用于设计 MISRA 规则的嵌入式系统; 它并不适合所有地方。（例如，编译器通常对符号表之类的东西使用动态内存分配，如果不是荒谬的话，没有动态内存分配将是困难的。

Most of these don't look to be on the main site anymore, and you can't browse that by subject anyway.  
    2007 年初[级 C](https://web.archive.org/web/20071105015557/http://www.accu.informika.ru/bookreviews/public/reviews/0sb/beginner_s_c.htm)（116 种）和 2008 年高级 [C](https://web.archive.org/web/20081020081215/http://www.accu.informika.ru/bookreviews/public/reviews/0sb/advanced_c.htm)（76 种）的 ACCU 审查书籍存档列表。其中大多数看起来不再出现在主网站上，而且无论如何你都无法按主题浏览。

Warnings  警告
============

There is a list of books and tutorials to be cautious about at the [ISO 9899 Wiki](http://iso-9899.info/wiki/Main_Page#Stuff_that_should_be_avoided), which is not itself formally associated with ISO or the C standard, but contains information about the C standard (though it hails the release of ISO 9899:2011 and does not mention the release of ISO 9899:2018).  
[ISO 9899 Wiki](http://iso-9899.info/wiki/Main_Page#Stuff_that_should_be_avoided) 上有一个需要谨慎的书籍和教程列表，它本身并不正式与 ISO 或 C 标准相关联，但包含有关 C 标准的信息（尽管它欢迎 ISO 9899：2011 的发布，并且没有提及 ISO 9899：2018 的发布）。

Be wary of books written by [Herbert Schildt](http://catb.org/%7Eesr/jargon/html/B/bullschildt.html). In particular, you should stay away from [C: The Complete Reference](https://rads.stackoverflow.com/amzn/click/com/0072121246) (4th Edition, 2000), known in some circles as C: The Complete Nonsense.  
警惕[赫伯特 · 希尔特](http://catb.org/%7Eesr/jargon/html/B/bullschildt.html)写的书。特别是，您应该远离 [C：完整参考（](https://rads.stackoverflow.com/amzn/click/com/0072121246) 第 4 版，2000 年），在某些圈子中被称为 C：完全废话。

Also do not use the book [Let Us C](https://smile.amazon.com/dp/9387284492/) (16th Edition, 2017) by Yashavant Kanetkar. Many people view it as an outdated book that teaches Turbo C and has lots of obsolete, misleading and incorrect material. For example, page 137 discusses the expected output from `printf("%d %d %d\n", a, ++a, a++)` and does not categorize it as [undefined behaviour](https://stackoverflow.com/questions/949433/why-are-these-constructs-using-pre-and-post-increment-undefined-behavior) as it should. It also consistently promotes unportable and buggy coding practices, such as using `gets`, `%[\n]s` in `scanf`, storing the return value of `getchar` in a variable of type `char` or using `fflush` on `stdin`.  
也不要使用 Yashavant Kanetkar 所著的《[Let Us C](https://smile.amazon.com/dp/9387284492/)》（第 16 版 ，2017 年）一书。许多人认为它是一本过时的书，教授 Turbo C，并且有很多过时、误导性和不正确的材料。例如，第 137 页讨论了预期的输出 `printf("%d %d %d\n", a, ++a, a++)` ，但没有将其归类为应有的未[定义行为](https://stackoverflow.com/questions/949433/why-are-these-constructs-using-pre-and-post-increment-undefined-behavior) 。它还始终如一地提倡不可移植和有缺陷的编码实践，例如在 `scanf` 中使用 `gets`、`%[\n]s`、将 `getchar` 的返回值存储在 `char` 类型的变量中或在 `stdin` 上使用 `fflush`。

[Learn C The Hard Way](https://learncodethehardway.org/c/) (2015) by Zed Shaw. A book with mixed reviews. [A critique of this book](http://hentenaar.com/dont-learn-c-the-wrong-way) by Tim Hentenaar:  
Zed Shaw 的《 [学习 C：艰难之路](https://learncodethehardway.org/c/) 》（2015 年）。一本评价褒贬不一的书。 蒂姆 · 亨特纳尔 （Tim Hentenaar） 对[这本书的评论](http://hentenaar.com/dont-learn-c-the-wrong-way) ：

> To summarize my views, which are laid out below, the author presents the material in a greatly oversimplified and misleading way, the whole corpus is a bundled mess, and some of the opinions and analyses he offers are just plain wrong. I've tried to view this book through the eyes of a novice, but unfortunately I am biased by years of experience writing code in C. It's obvious to me that either the author has a flawed understanding of C, or he's deliberately oversimplifying to the point where he's actually misleading the reader (intentionally or otherwise).  
> 总结一下我的观点，如下所述，作者以一种非常过于简单和误导性的方式呈现材料，整个语料库是一团糟，他提供的一些观点和分析完全是错误的。我试图通过新手的角度来看待这本书，但不幸的是，我对多年用 C 编写代码的经验有偏见。对我来说很明显，要么作者对 C 的理解有缺陷，要么他故意过度简化，以至于他实际上误导了读者（有意或无意）。

"Learn C The Hard Way" is not a book that I could recommend to someone who is both learning to program and learning C. If you're already a competent programmer in some other related language, then it represents an interesting and unusual exposition on C, though I have reservations about parts of the book. _Jonathan Leffler_  
《艰难地学习 C》不是一本我可以推荐给既学习编程又学习 C 的人的书。如果你已经是其他相关语言的称职程序员，那么它代表了对 C 的有趣和不寻常的阐述，尽管我对本书的某些部分持保留态度。 _乔纳森 · 莱夫勒_
